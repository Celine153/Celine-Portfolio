Description

This program aims to detect suspicious events, specifically, the sign-out of an account ID that has not previously signed in. The account IDs are represented as integer values ranging from 1 to 1000, and a sign-out event is indicated by an integer below 0. The program's output is the total count of suspicious account IDs.â€‹

Explanation

  1. Users need to input the number of integers to be processed.
  2. Users are then required to input integer values to indicate sign-in/sign-out events. These inputs will be stored in an array (the array variable is 'arr') for further processing.
  3. The program will allocate memory for a Boolean array to serve as markers when suspicious events occur. Initialize each array item with a 'false' value.
  4. Declare a variable called 'ans' to store the count of detected suspicious events.
  5. For each item in the 'arr' array:
    1. Check the type of activity (Sign in/Sign out).
    2. If the activity type is a sign-out, then:
      1. Check the 'test' array. Since the 'test' array was initialized as 'false', if the account hasn't signed in previously, the 'test' array value for that item will remain 'false'.
      2. If the value in the 'test' array at that index is 'false', it is considered a suspicious event, and the 'ans' count is incremented by 1.
    3. However, if the activity type is not a sign-out (i.e., the item in 'arr' has a value < 0), then:
      1. Update the 'test' array at the same index as the 'arr' item's value to 'true'. This indicates that the corresponding ID has signed in.
  6. Repeat steps 5a to 5c until all items in the 'arr' array have been checked.
  7. Display the total count of detected suspicious events.
